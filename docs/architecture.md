## 三层认知架构

Crablet 实现了受**人类双系统理论**启发的三层认知模型，通过**认知路由器**智能选择处理层级：

| 层级 | 名称 | 机制 | 适用场景 | 响应时间 |
|:---:|:---:|:---|:---|:---:|
| **System 1** | 直觉响应 | Trie 前缀树 + Levenshtein 模糊匹配 | 问候、帮助、状态查询 | **< 10ms** |
| **System 2** | 深度分析 | ReAct 推理引擎 + 中间件流水线 | 搜索、编码、文件操作、知识问答 | **2-10s** |
| **System 3** | 群体协作 | 异步 Swarm 消息传递 (Tokio Channel) | 深度研究、多步骤复杂任务 | **10s+** |

### System 1 - 直觉响应层
- **IntentTrie 前缀树**：O(L) 查找
- **Levenshtein 模糊匹配**：容忍拼写错误
- **命令规则引擎**：正则表达式匹配

### System 2 - 深度分析层
核心是一个完整的 **Thought → Action → Observation** ReAct 循环引擎，辅以可插拔中间件流水线：
`请求 → [Safety] → [CostGuard] → [SemanticCache] → [Planning] → [RAG] → [SkillContext] → ReAct 引擎`

### System 3 - 群体协作层
基于 Swarm 的深度研究与多 Agent 协作，包含 Swarm 协调器、研究代理和用户代理，支持点对点消息传递和广播机制。

## 多层记忆系统

```
┌────────────────────────────────────────────────────────────┐
│                        记忆架构                              │
├────────────────┬────────────────┬──────────────────────────┤
│   工作记忆      │    情节记忆     │      语义记忆             │
│  (Working)     │   (Episodic)   │     (Semantic)           │
│                │                │                          │
│  滑动窗口       │  SQLite 持久化  │   知识图谱               │
│  VecDeque      │  会话历史       │   Neo4j / SQLite         │
│  O(1) 访问     │  全文检索       │   关系推理               │
├────────────────┴────────────────┴──────────────────────────┤
│                  记忆整合器 (Consolidator)                   │
│          LLM 驱动的摘要生成 → 写入向量存储                    │
└────────────────────────────────────────────────────────────┘
```

- **工作记忆**：每会话独立的滑动窗口，基于 DashMap 实现无锁并发访问。
- **情景记忆**：基于 SQLite 的完整对话历史持久化。
- **语义记忆**：知识图谱存储实体关系（支持 Neo4j 和 SQLite 双后端）。
- **记忆整合器**：定期将对话历史通过 LLM 摘要后写入向量存储，实现长期记忆。
