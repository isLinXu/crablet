## 2. 发现的关键问题（P0 — 阻塞级）

### 2.1 前端构建完全失败 ❌

| 问题 | 影响 | 修复 |
|------|------|------|
| `typescript: ~6.0.2` 不存在 | `npm install` 失败 | → `~5.7.3` |
| `@types/node: ^25.5.0` 不存在 | 类型解析失败 | → `^22.15.3` |
| `@vitejs/plugin-react: ^6.0.1` 与 `vite@^7.3.1` 冲突 | Peer dependency 不兼容 | → `^4.4.1` |
| `@types/tesseract.js` 是弃用包 | 无类型定义 | 已移除 |
| `@types/dompurify` 是弃用包 | 冗余 | 已移除 |
| `eslint.config.js` 中 `reactHooks.configs.flat.recommended` 未定义 | Lint 崩溃 | 待修复（需降级 eslint-plugin-react-hooks） |

### 2.2 CI/CD 全部工作流失效 ❌

| 问题 | 影响 | 修复 |
|------|------|------|
| `actions/checkout@v6` 不存在 | 所有 6 个 workflow 无法运行 | → `v4`（25 处修改） |
| `actions/upload-artifact@v7` 不存在 | Coverage / Desktop 构建失败 | → `v4` |
| `docker/build-push-action@v7` 不存在 | 容器构建失败 | → `v6` |

### 2.3 桌面端图标缺失/分辨率不足 ⚠️

| 问题 | 影响 | 修复 |
|------|------|------|
| `icon.ico` 仅 16×16 + 32×32 | Windows 高 DPI 模糊 | 重新生成 16~256 全尺寸 |
| `binaries/` 仅 macOS ARM64 sidecar | 跨平台打包后无法启动 | 生成多平台编译脚本 |
| `tauri.conf.json` 未包含 `icon.png` | Linux 打包缺失图标 | 已添加 |
| `targets` 缺少 Linux 目标 | 无法生成 AppImage/DEB | 已添加 `appimage` + `deb` |

### 2.4 智能体核心功能空壳 ⚠️

| 问题 | 影响 | 修复 |
|------|------|------|
| `workflow/engine.rs` 直接返回 `completed` | 工作流编排无实际执行 | 提供 `engine_v2.rs`（DAG 拓扑排序 + 并行执行） |
| `meta_controller/optimizer.rs` 仅记录日志 | 元认知优化不落地 | 提供 `optimizer_v2.rs`（配置修改 + 快照回滚） |
| `system1.rs` 仅 4 条硬编码规则 | 无法动态扩展 | 提供 `system1_dynamic.rs`（运行时注册 + 上下文感知） |
| `swarm.rs` 超时硬编码 120s | 无动态调整 | 提供 `swarm_dynamic_timeout.rs`（p95 学习 + 负载加权） |

### 2.5 Docker 构建缺陷 ⚠️

| 问题 | 影响 | 修复 |
|------|------|------|
| `node:26-alpine` 不存在 | 容器构建失败 | → `node:22-alpine` |
| `cargo-chef:latest-rust-1.96` 不存在 | 缓存层失效 | → `latest-rust-1.85` |
| 未复制根 `Cargo.toml` / `Cargo.lock` | Workspace 依赖解析失败 | 已添加复制步骤 |
| `docker-compose` 命令已废弃 | 脚本警告 | → `docker compose` |

```bash

#   target/release/bundle/nsis/Crablet_0.1.0_setup.exe

#   target/release/bundle/deb/Crablet_*.deb
```

## 6. 后续优化优先级

### P0 — 立即执行（阻塞构建）

1. **前端依赖修复后运行构建验证**：
   ```bash
   cd frontend && npm install && npm run build && npm run lint && npm run test:ci
   ```
   注意：eslint-plugin-react-hooks 版本可能仍需降级至 `^5.2.0` 以兼容 ESLint 9 flat config。

2. **提交 Cargo.lock**：作为可执行程序，`Cargo.lock` 必须提交到版本控制，确保构建可复现。

3. **sqlx 主 crate 依赖修复**：`extern crate self as sqlx` 是反模式，建议显式添加 `sqlx = "0.8"` 依赖。

### P1 — 短期（1-2 周）

4. **工作流引擎替换**：将 `engine_v2.rs` 接入 `workflow/mod.rs`，实现真正的 DAG 执行。

5. **元认知优化器落地**：将 `optimizer_v2.rs` 接入 `meta_controller/mod.rs`，实现配置自动修改。

6. **System 1 动态化**：将 `system1_dynamic.rs` 接入 `cognitive/mod.rs`，支持运行时规则注册。

7. **Swarm 动态超时**：将 `swarm_dynamic_timeout.rs` 接入 `agent/swarm/executor.rs`，替换硬编码 120s。

8. **前端组件拆分**：`ChatWindow.tsx` (969 行) 和 `Canvas.tsx` (910 行) 需要拆分为更小的组件。

### P2 — 中期（1-2 月）

9. **多平台 sidecar 编译**：在 CI 中构建 `x86_64-apple-darwin`, `x86_64-pc-windows-msvc`, `x86_64-unknown-linux-gnu` 三个 sidecar 并发布到 Release Assets。

10. **前端体积优化**：`d3` (~300KB), `tesseract.js` (~10MB), `pdfjs-dist` (~2MB) 已配置 chunk 拆分，但 tesseract.js 应改为动态 import。

11. **安全增强**：`apiKey` 明文存储在 `localStorage` 中，应改为 `sessionStorage` 或加密存储。

12. **测试竞态修复**：集成测试中大量使用 `std::env::set_var`，并行运行会互相干扰。

### P3 — 长期（3-6 月）

13. **代码签名与公证**：配置 Apple Developer ID 证书 + Windows EV 代码签名证书。

14. **自动更新机制**：集成 Tauri 的 `updater` 插件，支持后台自动更新。

15. **性能分析**：使用 `cargo flamegraph` 和 Chrome DevTools 分析前后端热点。

16. **文档站点完善**：docs-site 使用 MkDocs，需补充 API 文档和架构图。
