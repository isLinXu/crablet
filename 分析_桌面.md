| 平台 | 预期文件名 | 实际存在 | 状态 |
|------|-----------|---------|------|
| macOS ARM64 (Apple Silicon) | `crablet-aarch64-apple-darwin` | ✅ **34.6 MB** | ✅ |
| macOS Intel (x86_64) | `crablet-x86_64-apple-darwin` | ❌ 缺失 | 🔴 **关键缺失** |
| Windows x86_64 | `crablet-x86_64-pc-windows-msvc.exe` | ❌ 缺失 | 🔴 **关键缺失** |
| Linux x86_64 | `crablet-x86_64-unknown-linux-gnu` | ❌ 缺失 | 🔴 **关键缺失** |
| Linux ARM64 | `crablet-aarch64-unknown-linux-gnu` | ❌ 缺失 | 🔴 **关键缺失** |

### 4.1 关键发现

**🔴 这是当前最大的跨平台风险。**

`binaries/` 目录下只有 **macOS ARM64** 的 sidecar 二进制。这意味着：

- **在 Intel Mac 上**: Tauri 构建时找不到 `crablet-x86_64-apple-darwin`，启动时 sidecar 定位失败，应用无法启动后端服务。
- **在 Windows 上**: `build-desktop.ps1` 脚本期望 `crablet-x86_64-pc-windows-msvc.exe`，但仓库中没有，即使编译后端成功，sidecar 不会自动进入 Tauri 包。
- **在 Linux 上**: 无任何 sidecar 二进制，无法打包。

### 4.2 修复建议

1. **macOS Universal Binary**: 构建 `x86_64` 和 `aarch64` 两个版本，然后用 `lipo` 合并为通用二进制，或分别放置两个文件。
2. **Windows Sidecar**: 在 Windows 环境或交叉编译工具链中编译 `crablet-x86_64-pc-windows-msvc.exe`，放入 `binaries/`。
3. **Linux Sidecar**: 使用 `cross` 或 GitHub Actions 编译 `x86_64-unknown-linux-gnu` 和 `aarch64-unknown-linux-gnu` 版本。
4. **CI/CD 自动化**: 在 `.github/workflows/build-desktop.yml` 中增加多平台编译矩阵，自动产出所有 sidecar 并放入 `binaries/`。

## 7. 打包优化建议

### 7.1 配置层优化

1. **动态版本同步** (`tauri.conf.json`):
   ```json
   "version": "../Cargo.toml"
   ```
   或
   ```json
   "version": "../package.json"
   ```
   避免版本漂移。

2. **添加 Linux 打包目标**:
   ```json
   "targets": ["dmg", "app", "nsis", "appimage", "deb"]
   ```
   同时加入 `icon.png` 到 `bundle.icon` 数组。

3. **设置 CSP**:
   ```json
   "csp": "default-src 'self'; connect-src 'self' http://127.0.0.1:18799;"
   ```

4. **NSIS 配置增强**:
   - 添加 `displayLanguageSelector: true`（如需要）
   - 配置 `installerIcon` 和 `installHeaderImage`

### 7.2 构建脚本优化

1. **macOS 脚本**:
   - 修复 DMG 文件名：`${APP_NAME}_${APP_VERSION}_${ARCH}.dmg`
   - 添加 `xcrun notarytool submit` 和 `xcrun stapler staple` 步骤（可选）
   - 支持 `universal` 模式构建（`cargo tauri build --target universal-apple-darwin`）

2. **Windows 脚本**:
   - 添加 `signtool sign /a /fd SHA256 /tr http://timestamp.digicert.com ...` 签名步骤
   - 支持 `msi` 构建（`--bundles msi`）

3. **新增 Linux 脚本** (`build-desktop-linux.sh`):
   ```bash
   #!/usr/bin/env bash
   # 使用 cross 编译 sidecar + Tauri 构建
   cross build --release -p crablet --target x86_64-unknown-linux-gnu
   cp target/x86_64-unknown-linux-gnu/release/crablet \
      desktop/binaries/crablet-x86_64-unknown-linux-gnu
   cd desktop && cargo tauri build --bundles appimage,deb
   ```

### 7.3 资源优化

1. **图标压缩**: `icon.png` (1024×1024) 为 879 KB，对于 1024×1024 PNG 而言偏大，可通过 `oxipng` 或 `pngquant` 压缩到 ~200 KB 而不损失视觉质量。
2. **ICO 重构**: 重新生成 `icon.ico` 包含全尺寸（16~256）。
3. **ICNS 检查**: `icon.icns` 为 2.0 MB，正常范围。确认 `icon.iconset` 与 `icon.icns` 内容一致。

## 10. 总体评估

| 维度 | 评分 | 说明 |
|------|------|------|
| Tauri 2 配置正确性 | ⭐⭐⭐⭐☆ (4/5) | 核心配置正确，但缺少 Linux 目标、版本硬编码、CSP 未设置 |
| 图标资源完整性 | ⭐⭐⭐⭐☆ (4/5) | 平台图标齐全，但 `icon.ico` 分辨率不足，且未加入 `icon.png` 到 bundle |
| 外部二进制引用 | ⭐⭐☆☆☆ (2/5) | 仅 macOS ARM64 有 sidecar，严重阻碍跨平台发布 |
| 构建脚本健壮性 | ⭐⭐⭐⭐☆ (4/5) | macOS/Windows 脚本完善，但缺少 Linux、无签名、DMG 文件名有 bug |
| 启动画面体验 | ⭐⭐⭐⭐⭐ (5/5) | `splash.html` 设计精美，动画流畅，API Key 配置交互完整 |
| 桌面入口逻辑 | ⭐⭐⭐⭐⭐ (5/5) | `main.rs` sidecar 管理、keyring 缓存、优雅退出均设计优秀 |
| **跨平台打包可行性** | ⭐⭐⭐☆☆ (3/5) | **当前仅 macOS ARM64 可完整打包发布**；Windows/Linux 需要补充 sidecar 和构建配置 |

### 核心结论

> **当前项目桌面端在 macOS Apple Silicon 上已达到可发布状态，但在 Windows 和 Linux 上因缺少 `crablet` sidecar 二进制而无法打包运行。** 图标资源方面，Windows ICO 文件分辨率不足，需重新生成。建议优先解决 sidecar 多平台编译和 `icon.ico` 重构，然后逐步完善 Linux 打包和代码签名。
