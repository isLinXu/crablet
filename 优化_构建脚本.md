## 修复文件清单

### 1. Dockerfile

#### 问题
- `FROM node:26-alpine` — Node.js 26 不存在，最新 LTS 是 22
- `FROM lukemathwalker/cargo-chef:latest-rust-1.96` — Rust 1.96 不存在，最新稳定版是 1.85
- Dockerfile 未复制根 `Cargo.toml` 和 `Cargo.lock` 到 planner 阶段，workspace 依赖解析可能失败

#### 修改内容
- ✅ `node:26-alpine` → `node:22-alpine`
- ✅ `lukemathwalker/cargo-chef:latest-rust-1.96` → `lukemathwalker/cargo-chef:latest-rust-1.85`
- ✅ Planner 阶段添加根 `Cargo.toml` 和 `Cargo.lock` 复制
  - 在 `/app` 目录下复制根 workspace 文件
  - 再进入 `/app/crablet` 执行 `cargo chef prepare`
- ✅ Builder 阶段添加根 `Cargo.toml` 和 `Cargo.lock` 复制
  - 使用 `cargo build --release -p crablet` 构建 workspace 中的指定包
- ✅ 修正 runtime 阶段 COPY 路径
  - `/app/crablet/target/release/crablet` → `/app/target/release/crablet`（workspace 构建目标目录在根目录下）

---

### 2. Justfile

#### 问题
- `up:` 目标使用 `docker-compose`（旧命令），应改为 `docker compose`
- `desktop-sidecar-copy` 硬编码 `aarch64-apple-darwin`，不支持跨平台构建
- 缺少 Windows 和 Linux 的 sidecar 复制任务

#### 修改内容
- ✅ `docker-compose up -d` → `docker compose up -d`
- ✅ `docker-compose down -v` → `docker compose down -v`
- ✅ `desktop-sidecar-copy` 改为自动检测平台（通过 `uname -s` 和 `uname -m`）
  - 支持 macOS (aarch64/x86_64)、Linux (x86_64/aarch64)、Windows (x86_64)
- ✅ 新增 `desktop-sidecar-copy-linux` 任务
- ✅ 新增 `desktop-sidecar-copy-windows` 任务
- ✅ 新增 `desktop-sidecar-copy-all` 任务（复制所有平台 sidecar）
